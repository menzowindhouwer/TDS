Use the Console buttons to compose an IPA string in the buffer area, then press Cut or Copy to put it on the system clipboard. You can then paste it into any text editor or other application that supports cutting and pasting.

Characters supported by your computer keyboard can also be entered by simply typing them. The Backspace and Delete keys can be used to edit the generated string.

Non-spacing diacritics (accents etc.) must be entered after the character they modify.

The console is extensible: The second and third tabs have space for user-selected symbols. You can define new buttons that insert any Unicode character (not just from the IPA subset). User-defined buttons will be remembered for future Console sessions.

Many fonts do not provide the full set of IPA symbols. The Console will try to select a complete font when it starts up. Use the list next to the character buffer to change the font of the text you type. Use the Settings tab to change the font used on the symbol buttons. If you don't have a suitable Unicode font, you can download fonts from the internet. Note that symbols will be correctly inserted even if you do not have a font that can display them. (They will appear correctly on another computer, or if you install a suitable font later).

The Import IPA tab can be used to import (convert to Unicode) text that was written using the older non-unicode fonts published by SIL, or other compatible fonts.

The Settings tab allows you to customize the operation of the program. Changes in the settings will be remembered for future Console sessions.

In the Settings tab, you can choose whether to compose characters and diacritics whenever Unicode provides a combined character. Combined characters look better, but the diacritic cannot be separated from the base character during later editing.

The IPA Console can stay resident as an icon in the system tray (for window managers that support it). Because of security restrictions, this feature is not available with the unsigned version of the Console (see here for more information).

Additional information can be found on the IPA Console homepage, at http://languagelink.let.uu.nl/tds/ipa/.


Release information
2009 january 21
Better support for Mac OS X 10.5 (Leopard). The preferred Arial Unicode MS font is called ArialUnicodeMS on a Mac. The IPA Console system tray functionality isn't (properly) supported on OS X.
2008 august 26
Upgraded to Unicode 5.1. This version of Unicode has an official codepoint for the labiodental flap (U+2C71), which is now used in the pulmonic consonants table. The look-alike has been added to the other symbols.
2007 may 10
On Mac OS X the IPA Console also renders properly. Please report if this update has negative effects on other platforms.
2007 may 8
The default is now to put the Unicode text in all formats on the clipboard. The known word processors pick the most proper format by default. There is still an option on the settings tab to put just plain text on the clipboard.
2007 april 19
The clipboard format setting drop down has been simplified. Persistent settings have been made independent of the actual labels used in the settings tab.
2007 march 20
Finetuned the HTML clipboard format so it doesn't append a superfluous newline. Unfortunately Microsoft Word 2000 still needs the <tt/> wrapper to listen to all the font information in all cases. Added shortcut entries to the clipboard format settings for tested word processors (Microsoft Word 2000, 2003, WordPad and OpenOffice.org Writer).
2007 march 15
Microsoft Word 2000 doesn't listen to the RTF font information, so the default clipboard format is now HTML which works in all (tested) cases.
